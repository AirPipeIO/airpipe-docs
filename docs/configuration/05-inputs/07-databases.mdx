# Databases

:::tip Full field reference
Every database and `document_operation` field is in the
**[Databases reference](/docs/reference/inputs-database)**, generated from the engine schema.
:::

The database can be defined in the globals part of a configuration or the action itself.
Defining it in the globals section allows any action to utilise the database.
Any database config defined with a matching name will have its existing connection reused.

## Options

Connect with a `driver` + [`conn_string`](#connection-strings), or with discrete
`user` / `pass` / `host` / `dbname` fields — don't mix the two. Supported drivers:
`postgres`, `mysql`, `mssql`, `sqlite`, `mongodb`. The full field list is in the
[Databases reference](/docs/reference/inputs-database).

## document_operation {#document_operation}

NoSQL / document stores (MongoDB, Azure Cosmos DB, Amazon DocumentDB) use a
`document_operation` block instead of `query`: `database`, `collection`,
`operation` (`find`, `findOne`, `insertOne`, `updateMany`, `aggregate`, …) and
the relevant `filter` / `pipeline` / `insert` / `update` / `delete` / `options`
(each a JSON string). See the
[document_operation reference](/docs/reference/inputs-database#type-documentoperation).

## Connection Strings

Below are example connection strings, please adjust the parameters to your own requirements.

- ### Postgres
    - #### CockroachDB
    - #### YugabyteDB
    - #### Timescale
    - #### Any postgres compatible databases

      ```
      postgresql://username:password@host:port/dbname[?paramspec]
      ```
    
- ### MySQL
    - #### MariaDB
    - #### TiDB
    - #### Any mysql compatible database

      ```
      mysql://root:college@localhost:3306/employees
      ```

- ### SQLite

      ```
      sqlite:///home/my-user/db-files/Sqlite-Car_Database.db
      ```

- ### MSSQL / SqlServer

      ```
      jdbc:sqlserver://localhost:1433;databaseName=AdventureWorks;user=SA;password=Passw0rd;trustServerCertificate=true;
      ```

- ### MongoDB
    - #### Azure Cosmos DB
    - #### Amazon DocumentDB

## Global Example

```yml
name: database
version: 0.0.1

  databases:
    main:
      driver: postgres
      conn_string: |
        postgresql://a|env::POSTGRES_USER|:a|env::POSTGRES_PASS|@a|env::POSTGRES_HOST|?connect_timeout=10"
```

## Action Example

**Note** the `query` option is for relational based database, and not used for NoSQL/Document Store type databases such as MongoDB, Azure Cosmos DB, Amazon DocumentDB.

```yml
name: database
version: 0.0.1

interfaces:
  database/example:
    output: http
    method: POST

    actions:
      - name: UserList
        database:
          dbname: main
          driver: postgres
          conn_string: |
            postgresql://a|env::POSTGRES_USER|:a|env::POSTGRES_PASS|@a|env::POSTGRES_HOST|?connect_timeout=10"
        query: |
          SELECT * FROM Users;
```
