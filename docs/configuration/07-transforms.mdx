# Transforms & Functions

Are an **array of functions** which are used to transform your data after it is received. To use **transform functions** you must declare a `post_transforms` block in your specified **action**

```yaml
name: json-tests
metrics_enabled: true
docs: true

interfaces:
  tests/add_attribute:
    output: http
    method: GET

    actions:
      - name: AddAttribute
        http:
          url: https://jsonplaceholder.typicode.com/todos/10
        post_transforms:
          - extract_value: ".body"
          - add_attribute:
              env: "production"
```

## Available transforms

Air Pipe ships a large set of transform functions. The **[Transforms
reference](/docs/reference/transforms)** documents every one with its fields and
an example — it is generated directly from the engine schema, so it never drifts.

Common ones:

| Transform | Does |
|---|---|
| `extract_value` / `extract_with_jq` / `extract_with_regex` | Pull a value or subset out of the data |
| `add_attribute` / `rename_keys` / `remove_keys` | Reshape objects |
| `keep_attributes` / `remove_attributes` / `move_attributes` | Allow/deny/relocate fields |
| `filter` | Keep only elements matching [assertion conditions](/docs/configuration/interfaces/actions/asserts) |
| `group_by` / `numerics` / `math` | Aggregate and compute |
| `generate_password` / `generate_bytes` / `encrypt_value` | Secrets & crypto |
| `add_jwt` / `bcrypt` | Auth helpers |
| `nested_transforms` | Apply transforms per key |

See the full list and fields in the [Transforms reference](/docs/reference/transforms).

## Add JWT {#add-jwt}

`add_jwt` signs a JWT from your data and adds it under a key. See the
[AddJWT reference](/docs/reference/transforms#type-addjwt) for all fields and an
example.
